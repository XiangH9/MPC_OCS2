# 启动说明

本文档说明如何在当前 `uni_ws_snapshot` 工作区中完成编译与启动。


1. Ubuntu 版本
2. ROS 2 版本
3. Gazebo 版本
4. OCS2 源码依赖是否在同一个工作区
5. `quadruped_ros2_control` 是否和 `ocs2_ros2` 一起编译


建议尽量按照下面环境复现：

- Ubuntu 22.04
- ROS 2 Humble
- Gazebo Harmonic
- 工作区中同时包含：
  - `src/ocs2_ros2`
  - `src/quadruped_ros2_control`
  - `src/unitree_sdk2`


在当前工作区中，**不能只编译控制器本身**。

至少要保证下面两部分已经一起编译：

1. `ocs2_ros2`
2. `quadruped_ros2_control`

原因是：

- `unitree_guide_controller` 直接依赖 OCS2 的多个包，例如：
  - `ocs2_core`
  - `ocs2_mpc`
  - `ocs2_sqp`
  - `ocs2_centroidal_model`
  - `ocs2_pinocchio_interface`
  - `ocs2_legged_robot`

所以如果只编译 `quadruped_ros2_control`，而工作区里没有一起编译 `ocs2_ros2`，当前 NMPC 控制器是无法正常编译和运行的。


先进入工作区根目录：

```bash
cd ~MPC_OCS2
```

加载 ROS 2 环境：

```bash
source /opt/ros/humble/setup.bash
```

如果你自己使用 conda 环境，可以再激活你自己的环境，例如：

```bash
conda activate uros
```

或者你自己的其他环境名：

```bash
conda activate my_env
```

然后先安装依赖：

```bash
rosdep install --from-paths src --ignore-src -r -y
```

## 5. 推荐的完整编译命令

最稳妥的是直接在整个工作区下编译：

```bash
colcon build --symlink-install
```

如果你只想先编译当前 NMPC 与 Gazebo 仿真最相关的内容，建议至少执行：

```bash
colcon build --packages-up-to \
  unitree_guide_controller \
  gz_quadruped_hardware \
  gz_quadruped_playground \
  keyboard_input \
  go1_description \
  --symlink-install
```

注意：

- 这条命令虽然只写了几个包
- 但前提仍然是 `src/ocs2_ros2` 已经在工作区里
- `colcon` 会自动把这些包依赖到的 OCS2 包一起编译



启动控制器与仿真：

```bash
ros2 launch unitree_guide_controller gazebo_classic.launch.py pkg_description:=go1_description rviz:=false
```

启动键盘输入：

```bash
ros2 run keyboard_input keyboard_input
```
